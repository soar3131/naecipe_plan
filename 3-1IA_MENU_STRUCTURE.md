## 1. 서비스 전체 구조 개요

**플랫폼 공통 구조**

* 상단 글로벌 헤더(웹 / 하이브리드 앱 공통)

  * 로고(홈 이동)
  * 통합 검색 바
  * 레시피북 진입 아이콘
  * 마이(로그인/계정) 아이콘 
* 주요 사용자 여정(Core Loop)

  * 검색 → 레시피 리스트 → 레시피 상세
    → 레시피로 조리 → 피드백 입력 → AI 보정
    → 보정 레시피를 내 레시피북에 저장

Phase 1에서는 **검색/상세/레시피북/AI 보정/Q&A**에 집중하고,
피드·유저 레시피 작성/커머스 등은 Phase 2 이후로 분리합니다.

---

## 2. 최상위 메뉴 구조 (Phase 1 기준)

### 2-1. 최상위 메뉴 트리

```text
[Global Shell]
 ├─ 홈 (H-01)
 │   ├─ 최근 검색어 블록 (H-03, Should)
 │   ├─ 최근 본 레시피 블록 (H-03, Should)
 │   └─ (향후) 추천 레시피 / 개인화 영역
 │
 ├─ 검색 (G-01 / S-01)
 │   ├─ 검색 입력 (G-01, H-02)
 │   ├─ 검색 결과 리스트 (S-01)
 │   │   ├─ 정렬 옵션 (S-02)
 │   │   └─ 필터 영역 (조리 시간/난이도/맛·대상 태그) (S-03)
 │   └─ (빈 결과 시) 대체 추천/안내
 │
 ├─ 레시피 상세 (R-01)
 │   ├─ 기본 정보 영역 (제목/출처/썸네일/시간/난이도/인분) (R-01)
 │   ├─ 재료 섹션 (R-02)
 │   ├─ 조리 단계 섹션 (R-03)
 │   ├─ 스코어/태그 영역 (R-04)
 │   ├─ 액션 영역
 │   │   ├─ 레시피북에 저장 (R-05)
 │   │   └─ 이 레시피로 요리 시작 (R-05)
 │   └─ Q&A 영역 (Q-01)
 │
 ├─ 조리/피드백/AI 보정 플로우
 │   ├─ 요리 시작 모달 (C-01)
 │   ├─ 조리 완료 & 피드백 입력 모달 (C-02)
 │   ├─ AI 보정 진행/결과 화면 (A-01)
 │   └─ 원본 vs 보정 비교 뷰 (A-02, Should)
 │
 ├─ 내 레시피북 (B-01)
 │   ├─ 레시피북 리스트 (B-01)
 │   │   ├─ 원본만 저장된 레시피
 │   │   └─ 보정 버전 포함 레시피(뱃지 표기)
 │   ├─ 레시피북 상세(원본/보정 묶음 보기) (B-02)
 │   └─ 폴더/라벨(카테고리) 관리 (B-03, Should)
 │
 ├─ Q&A 에이전트 (Q-01, Q-02)
 │   └─ 레시피 상세 하단 내 임베디드 모듈
 │       ├─ 카테고리 선택(재료 대체/보관/도구/기타)
 │       ├─ 질문 입력
 │       └─ 답변 표시 및 만족도 피드백(선택)
 │
 └─ 계정/세션 (A-U-01, A-U-02)
     ├─ 로그인/회원가입 진입
     ├─ 비로그인 상태에서 저장/보정 시 계정 생성 유도
     └─ (향후) 마이 페이지/설정
```

위 구조는 기능 명세서에서 정의한 화면 ID(G-01, H-01, S-01 등)를 기준으로 정리한 IA이며,
Phase 1 Must/Should 범위와도 1:1로 매핑됩니다.

## 3. 주요 화면·메뉴별 상세 IA

### 3-1. Global Shell / 공통

* **G-01 글로벌 헤더**

  * 로고 → 홈(H-01)
  * 검색 입력 → 검색 결과(S-01)
  * 레시피북 아이콘 → 내 레시피북(B-01)
  * 마이 아이콘 → 로그인/계정(A-U-01) 
* **G-02 공통 토스트**

  * 저장 성공/실패, AI 보정 실패, Q&A 한도 초과 등 공통 알림 레이어

---

### 3-2. 홈 & 검색

**(1) 홈(H-01)** 

* 최근 검색어 블록(H-03, Should)
* 최근 본 레시피 블록(H-03, Should)
* (향후) 추천 레시피 영역 – 개인화 추천 기능과 연동

**(2) 검색**

* 헤더 검색 바(G-01, H-02)

  * 메뉴/재료/상황 키워드 기반 검색(예: 김치찌개, 닭가슴살, 아이 반찬)
* 검색 결과 리스트(S-01)

  * 정렬: 종합 스코어(기본) / 최신 / 조리 시간 / 난이도 (S-02)
  * 필터: 조리 시간, 난이도, 맛/대상 태그(가능 시) (S-03)
  * 각 카드: 썸네일, 제목, 출처, 시간/난이도, 핵심 태그, 신뢰도 스코어 표시

---

### 3-3. 레시피 상세

**레시피 상세(R-01~R-05)**

* 상단

  * 제목, 출처 타입(셰프/인플루언서/블로그/유저 등), 채널명, 원본 링크
* 기본 정보

  * 대표 이미지, 예상 조리 시간, 난이도, 기준 인분
  * 신뢰도/인기도 스코어(종합 점수 + SNS/방송/내부 지표 요약)
* 재료 섹션 (R-02)

  * 재료명·양·단위 리스트
* 조리 단계 섹션 (R-03)

  * 단계 번호별 설명 (텍스트 중심, 이미지/영상은 선택)
* 태그 섹션 (R-04)

  * 맛(매콤/담백 등), 대상(아이용/다이어트 등), 스타일/재료 태그 
* 액션 영역 (R-05)

  * [레시피북에 저장]
  * [이 레시피로 요리 시작] (조리/피드백 플로우로 진입)

---

### 3-4. 조리 기록 & 피드백 / AI 보정

**(1) 요리 시작 모달 – C-01**

* 인분 수 선택 → `cooking_started` 로그 기록 후 상세 화면 유지

**(2) 조리 완료 & 피드백 – C-02**

* 간/맛/매운맛/난이도에 대한 선택형 평가 + 자유 코멘트
* “이 피드백을 바탕으로 보정하기” 체크박스(기본 ON)
* [피드백 저장 & 보정 요청] → AI 보정(A-01) 호출

**(3) AI 보정 결과 – A-01**

* 보정 진행 로딩 상태 → 보정 레시피 생성
* 주요 변경점 요약(재료 비율/조리 시간 변화 등)
* [보정 레시피 상세 보기] / [레시피북에 저장] 버튼
* 저장 시 원본–보정 묶음 구조로 레시피북(B-01/02)에 반영

**(4) 원본 vs 보정 비교 – A-02 (Should)**

* 재료/시간 비교 테이블
* 변경된 항목 하이라이트

---

### 3-5. 내 레시피북

**레시피북 리스트 / 상세 – B-01, B-02, B-03**

* 레시피북 리스트(B-01)

  * 저장일 기준 정렬
  * “보정 버전 있음” 뱃지로 구분
* 레시피북 상세(B-02)

  * 원본/보정 탭 or 토글
  * 보정 레시피에서 바로 조리 시작/다시 보정 가능(향후)
* 폴더/카테고리(B-03, Should)

  * 1단계 라벨 구조(예: 평일 저녁/아이 반찬/손님 초대 등)

---

### 3-6. Q&A 에이전트

**레시피 상세 내 Q&A 영역 – Q-01, Q-02**

* 카테고리 선택: 재료 대체 / 보관 / 도구 / 기타
* 질문 입력 → 내부 전문 요리 문서 + 레시피 문맥 기반 RAG 구조로 답변 생성
* 위험/보관 관련 질문에는 보수적인 안전 가이드 포함
* 일일 호출 한도 초과 시 안내 메시지

---

### 3-7. 계정/세션

**A-U-01, A-U-02 (MVP 최소 계정 구조)**

* 비로그인:

  * 검색, 상세, Q&A 일부 사용 가능
  * 레시피북 저장/보정 레시피 저장 시 로그인 유도
* 로그인:

  * 레시피북/보정 레시피/취향 데이터가 user_id 기준으로 축적

## 4. 주요 화면 간 이동 관계 요약

PLAN 가이드에서 요구한 “화면 간 이동 관계”를 IA 관점에서 단순화하면 다음과 같습니다.

1. **홈(H-01) → 검색 결과(S-01)**

   * 검색 바에 키워드 입력 + 엔터/검색 버튼

2. **검색 결과(S-01) → 레시피 상세(R-01)**

   * 레시피 카드 탭

3. **레시피 상세(R-01) → 조리/피드백/보정(C-01 → C-02 → A-01/A-02)**

   * [이 레시피로 요리 시작] → C-01
   * “조리 완료/피드백 남기기” → C-02
   * [피드백 저장 & 보정 요청] → A-01 → (선택) A-02

4. **레시피 상세(R-01) → Q&A(Q-01)**

   * 상세 하단 Q&A 영역에서 질문 → Q&A 응답 표시

5. **어디서나(Global 헤더) → 레시피북(B-01)**

   * 헤더의 레시피북 아이콘 클릭

6. **레시피북(B-01) → 레시피북 상세(B-02)**

   * 리스트 아이템 클릭
   * 보정 버전 있는 경우, 보정 레시피 탭 우선 노출(정책)

7. **비로그인 상태에서 저장/보정 시 → 로그인/회원가입(A-U-01)**

   * 로그인/회원가입 완료 후 원래 진입 화면으로 복귀
