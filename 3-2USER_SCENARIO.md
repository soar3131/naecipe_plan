## 1. 문서 목적 & 범위

### 1.1 목적

이 문서는 **내시피 Phase 1 MVP**에서 사용자가 실제로 어떤 흐름으로 서비스를 이용하는지, **사용자 경험 관점**에서 스토리 형태로 정리한다.

* 상위 기획서와 PRD에서 정의한 **Core Loop**

  > 검색 → 레시피 상세 → 조리/사용 → 피드백 입력 → AI 보정 → 보정 레시피를 레시피북에 저장  
* 요구사항 정의/기능 명세에서 정의한 **세부 기능**을
  “사용자가 무엇을 하고 싶어 하는지” 관점의 **User Story**로 재구성하기 위함이다.  

### 1.2 범위

* **포함 (Phase 1 Must/Should)** 

  * 통합 레시피 검색 & 리스트
  * 레시피 상세 보기
  * 레시피북 저장/조회
  * 조리 기록(요리 시작/완료)
  * 피드백 입력 및 AI 보정 레시피 생성
  * Q&A 요리 에이전트
  * 기본 계정/세션, 핵심 로깅

* **참고로만 포함 (향후 Phase 2+)** 

  * 유저 레시피 게시 및 피드
  * 크리에이터 채널/대시보드

## 2. 주요 페르소나 요약

서비스 기획서·PRD에서 정의한 3개 핵심 페르소나를 기준으로 한다.  

1. **P1 – 집밥러 (초·중급 가정 요리사)**

   * 20~40대 직장인/주부/자취생
   * “오늘 뭐 해 먹지?”가 일상, 실패하기 싫고 **맛 보장**을 중요하게 생각
   * 짧은 시간, 적당한 난이도, 내 입맛(짠/단/매운 정도)에 맞는 레시피를 원함

2. **P2 – 레시피 크리에이터 (셰프/인플루언서/블로거)**

   * 여러 채널에 이미 레시피를 올리고 있음
   * 내 레시피가 **어떤 유저에게, 어떤 상황에서** 쓰이는지 알고 싶고
   * 데이터 기반 인사이트와 팬과의 새로운 접점을 원함
   * *Phase 1에서는 주로 “레시피 출처”로만 등장, 적극적 활동 시나리오는 Phase 2.*

3. **P3 – 요리 파워 유저·취미 요리인**

   * 새로운 메뉴/세계 요리에 관심 많은 상급 유저
   * 태그·필터를 세밀하게 쓰며, 다양한 버전의 레시피 비교·튜닝을 즐김
   * “남들이 만든 보정 레시피”에도 관심이 많음 (Phase 2에서 본격화)

## 3. 핵심 사용자 시나리오 (Flow 중심)

### SC-01. 오늘 저녁 메뉴를 찾고, 한 번에 Core Loop 완주하기 (P1 / Phase 1 핵심)

**목표**

* 오늘 저녁 메뉴를 빠르게 정하고,
* **신뢰도 높은 레시피**를 선택해 실패 없이 요리한 뒤,
* 내 입맛에 맞게 AI 보정된 레시피를 **레시피북에 저장**한다. 

**사전 조건**

* 사용자는 회원 가입을 완료했고 로그인 상태(P1에서는 레시피북 저장 시점에 가입 유도 가능). 

**주 흐름**

1. 홈/헤더 검색바에 `"김치찌개"` 입력 후 검색
2. 검색 결과 화면에서

   * 상단에 `“'김치찌개' 검색 결과 123건”` 표시
   * 기본 정렬은 **종합 스코어 순** (신뢰도/인기도 기준) 
3. 사용자는 필터에서

   * 조리 시간: `30분 이내`
   * 난이도: `초급`
     을 선택해 결과를 좁힌다. 
4. 카드 리스트에서

   * 썸네일, “아이도 좋아하는”, “담백한” 태그, 높은 스코어를 가진 레시피를 선택해 상세 진입
5. 레시피 상세 화면에서

   * 재료/분량, 조리 시간(예: 25분), 난이도, 태그(담백한, 아이도 좋아하는)를 확인
   * “레시피북에 저장”을 눌러 기본 레시피를 저장
   * “이 레시피로 요리 시작”을 눌러 인분 수(예: 3인분)를 선택하고 조리 시작으로 기록된다. 
6. 실제로 요리 후, 상세 화면에서 “조리 완료 & 피드백 남기기” 버튼을 누른다.
7. 피드백 모달에서

   * 간: `조금 짰다`
   * 맛/풍미: `적당하다`
   * 매운맛: `거의 안 맵다`
   * 난이도: `보통`
   * 코멘트: “아이에게 주기엔 간이 조금 센 느낌”
     을 입력하고 “피드백 저장 & 보정 요청”을 선택. 
8. AI 보정 화면에서

   * “간을 줄이고 아이에게 더 적합하게 조정했어요.”라는 요약과 함께
   * 간장/소금 양이 20~30% 줄어든 보정 레시피가 생성된다. 
9. 사용자는 보정된 레시피를 확인한 뒤,

   * “보정 레시피를 레시피북에 저장”을 선택
   * 레시피북에는 `[원본 김치찌개] + [내 입맛에 맞게 보정된 김치찌개 v1]` 묶음으로 관리된다. 

**성공 조건**

* 사용자가 한 세션 내 혹은 일정 기간 안에
  “검색 → 레시피 상세 → 조리 → 피드백 → AI 보정 → 레시피북 저장”을 **1회 이상 완주**.

### SC-02. 레시피북에서 보정 레시피를 다시 사용하기 (P1 / 재사용 검증)

**목표**

* 이전에 보정해 둔 레시피를 다시 사용해서
* 반복 요리 시에도 일관된 맛을 얻고,
* 보정 레시피의 **재사용률**을 높인다.  

**사전 조건**

* SC-01과 같이 김치찌개 보정 레시피 v1이 레시피북에 저장되어 있음.

**주 흐름**

1. 일주일 뒤, 사용자가 내시피 접속 → 상단에서 “레시피북”으로 이동
2. “내 레시피북” 리스트에서

   * “내 보정 김치찌개” 항목을 선택 (뱃지: `보정 버전 있음`) 
3. 레시피북 상세(비교 뷰)에서

   * 기본 탭은 “보정 레시피”로 열리며,
   * 상단에서 “원본 / 보정” 전환 가능
   * 재료 테이블에서 원본과 보정의 양/비율 차이가 하이라이트된 상태로 노출(Should). 
4. 사용자는 보정 레시피를 기준으로 “이 레시피로 요리 시작”을 눌러 다시 조리
5. 조리 후 간단 피드백(예: “이번엔 딱 좋았어요”)을 남기고, 추가 보정 없이 완료

**성공 조건**

* 보정 레시피를 기준으로 `cooking_started`/`adjusted_recipe_used` 이벤트가 쌓인다. 
* 동일 사용자가 보정 레시피를 **2회 이상 사용**하면 “보정 레시피 재사용률” 지표에 반영.

### SC-03. 요리 중 궁금한 점을 Q&A로 해결하기 (P1)

**목표**

* 레시피를 보며 요리하다가 떠오르는
  “대체 재료, 보관법, 도구 사용법” 같은 질문을
  화면 이탈 없이 해결한다. 

**사전 조건**

* 사용자는 레시피 상세 화면에 진입해 조리 중.

**주 흐름**

1. 사용자가 레시피 상세 화면을 보다가 재료 리스트를 보고 생각한다.

   * “대파가 없는데, 뭘 대신 넣을 수 있지?”
2. 상세 하단의 Q&A 영역에서 카테고리 `재료 대체` 선택 후

   * 질문 입력: “대파 대신 넣을 수 있는 재료가 있을까요?”
3. “보내기” 버튼을 누르면

   * 레시피 ID + 질문 + 카테고리 정보와 함께 Q&A 에이전트 호출
4. 잠시 후, 응답 영역에

   * “대파 대신 양파의 흰 부분을 얇게 썰어 사용하거나, 쪽파를 사용해도 됩니다. 다만 향의 강도가 조금 다르니 양을 70~80% 정도로 줄여 사용해 보세요.”
     와 같은 답변이 노출된다. 
5. 사용자는 안내에 따라 재료를 대체해서 조리를 이어간다.

**예외 흐름**

* 일일 Q&A 호출 한도 초과 시

  * “오늘 이용 가능한 Q&A 횟수를 모두 사용했습니다.” 안내 후 추가 질문 차단. 
* 모델 오류/타임아웃 시

  * “답변을 가져오지 못했습니다. 잠시 후 다시 시도해 주세요.” 메시지 노출.

**성공 조건**

* 사용자가 레시피 상세 화면에서 이탈하지 않고,
  필요한 Q&A를 통해 조리를 계속 진행.
* `qa_asked` 이벤트 및 카테고리 정보 수집.

### SC-04. 비로그인 사용자의 첫 레시피북 저장 & 가입 유도 (P1)

**목표**

* 게스트 유저가 레시피북 저장/AI 보정 기능을 쓰려다
  자연스럽게 **회원 가입/로그인**으로 유도된다. 

**주 흐름**

1. 게스트 유저가 검색 → 레시피 상세까지 진입
2. “레시피북에 저장” 버튼 클릭
3. 시스템이 로그인 상태가 아님을 확인하고

   * 모달: “내 레시피북에 저장하려면 로그인이 필요합니다.”
   * 버튼: [로그인] [회원가입] [취소] 
4. 사용자가 [회원가입] 선택 → 최소한의 가입 플로우(이메일/소셜 등) 완료
5. 가입 완료 후,

   * 원래 보고 있던 레시피 상세 화면으로 복귀
   * 자동으로 레시피북 저장을 재시도하거나, “레시피북에 저장 완료” 토스트 표시

**성공 조건**

* 게스트 상태에서 시작했으나

  * 1. 계정을 생성하고
  * 2. 최소 1건의 레시피를 레시피북에 저장

## 4. User Story 목록 (Epic 별 정리)

> 각 User Story는 “**As a / I want / So that**” 구조를 기본으로 하고,
> Phase(=우선순위)와 연결 기능을 함께 표시한다.

### 4.1 Epic – 통합 검색 & 탐색

**E-SEARCH**

1. **US-SEARCH-01 (P1 / Must)**

   * As a **집밥러**,
   * I want to **메뉴명, 재료명, 상황 키워드로 레시피를 검색**할 수 있고,
   * so that **오늘 내 상황(아이 반찬, 다이어트 등)에 맞는 레시피를 빠르게 찾을 수 있다.** 
   * *AC 힌트*: 검색어 2자 이상에서 실행 가능, 결과 수/빈 상태 메시지 표시. 

2. **US-SEARCH-02 (P1 / Must)**

   * As a user,
   * I want **검색 결과가 신뢰도/인기도 스코어 순으로 정렬된 기본 리스트**를 보고,
   * so that **무엇을 믿고 따라야 할지** 쉽게 판단할 수 있다. 

3. **US-SEARCH-03 (P1 / Must)**

   * As a 집밥러,
   * I want to **조리 시간/난이도 필터**를 사용해 결과를 좁히고,
   * so that **나의 시간과 실력에 맞는 레시피만** 볼 수 있다. 

4. **US-SEARCH-04 (P1 / Should)**

   * As a 집밥러,
   * I want to **최근 검색어/최근 본 레시피를 홈에서 바로 다시 실행**하고,
   * so that **자주 찾는 레시피로 빠르게 돌아갈 수 있다.** 

5. **US-SEARCH-05 (P1 / Must)**

   * As a 사용자,
   * I want **내 검색어가 남에게 노출되지 않고 안전하게 저장**되길 원하며,
   * so that **프라이버시를 걱정하지 않고 검색할 수 있다.** 

---

### 4.2 Epic – 레시피 상세 & 조리

**E-DETAIL**

1. **US-DETAIL-01 (P1 / Must)**

   * As a 집밥러,
   * I want **레시피 제목, 출처, 재료, 분량, 조리 단계, 시간, 난이도, 태그를 한 화면에서 명확히 보고**,
   * so that **다른 사이트를 왔다 갔다 하지 않고도 조리할 수 있다.** 

2. **US-DETAIL-02 (P1 / Must)**

   * As a user,
   * I want **이 레시피로 요리를 시작했다는 것을 표시**하고,
   * so that **나의 조리 기록과 이후 피드백/보정 과정이 하나의 흐름으로 관리**되길 원한다. 

3. **US-DETAIL-03 (P1 / Must)**

   * As a user,
   * I want to **레시피 상세에서 바로 레시피북에 저장**하고,
   * so that **좋았던 레시피를 잊지 않고 나만의 컬렉션으로 관리**할 수 있다. 

4. **US-DETAIL-04 (P1 / Should)**

   * As a user,
   * I want **스코어(신뢰도/인기도)를 요약과 간단한 세부 지표로 확인**하고,
   * so that **이 레시피가 어느 정도 검증된 레시피인지 직관적으로 알 수 있다.** 

---

### 4.3 Epic – 레시피북 저장 & 관리

**E-BOOK**

1. **US-BOOK-01 (P1 / Must)**

   * As a 로그인된 사용자,
   * I want **관심 있는 레시피를 레시피북에 저장**하고,
   * so that **나만의 요리 컬렉션을 만들 수 있다.** 

2. **US-BOOK-02 (P1 / Must)**

   * As a user,
   * I want **레시피북 리스트에서 내 레시피들을 한 번에 보고**,
   * so that **다음 번 요리 때 쉽게 다시 찾아 쓸 수 있다.** 

3. **US-BOOK-03 (P1 / Should)**

   * As a 집밥러,
   * I want **“평일 저녁”, “아이 반찬” 등 라벨/폴더로 레시피북을 분류**하고,
   * so that **상황별로 바로 꺼내 쓸 수 있다.** 

4. **US-BOOK-04 (P1 / Must)**

   * As a user,
   * I want **보정 레시피가 존재하는 레시피를 명확히 구분해서 표시**해 주길 원하며,
   * so that **원본 대신 내 입맛에 맞게 보정된 버전을 우선 사용할 수 있다.** 

---

### 4.4 Epic – 피드백 & AI 레시피 보정

**E-ADJUST**

1. **US-ADJ-01 (P1 / Must)**

   * As a 집밥러,
   * I want **요리를 해본 뒤 간, 맛, 매운맛, 난이도에 대한 피드백을 간단한 선택형으로 남길 수 있고**,
   * so that **AI가 이 정보를 바탕으로 내 입맛을 학습하고 레시피를 조정**해 주길 원한다.  

2. **US-ADJ-02 (P1 / Must)**

   * As a user,
   * I want **“조금 짰다/너무 맵다” 같은 피드백을 주면 재료 비율과 조리 시간을 조정한 보정 레시피가 자동 생성**되길 원하며,
   * so that **두 번째부터는 실패 확률이 낮아지고 내 입맛에 더 맞는 결과를 얻을 수 있다.** 

3. **US-ADJ-03 (P1 / Must)**

   * As a user,
   * I want **보정 레시피를 원본과 나란히 비교**하고,
   * so that **어떤 재료가 얼마나 바뀌었는지 이해한 후 안심하고 사용할 수 있다.**  

4. **US-ADJ-04 (P1 / Must)**

   * As a user,
   * I want **보정 레시피를 레시피북에 저장**하고,
   * so that **다음 번에는 이 버전을 기본 레시피처럼 사용할 수 있다.** 

---

### 4.5 Epic – 요리 Q&A 에이전트

**E-QNA**

1. **US-QNA-01 (P1 / Must)**

   * As a 집밥러,
   * I want **레시피 상세 화면을 보면서 재료 대체/보관/도구 사용에 대해 바로 질문**하고,
   * so that **검색창을 따로 열지 않고도 요리 흐름을 유지**할 수 있다. 

2. **US-QNA-02 (P1 / Must)**

   * As a user,
   * I want **식품 안전/보관 관련 질문에 대해서는 항상 보수적이고 안전한 답변을 받고**,
   * so that **위험한 정보를 따라 하지 않도록 보호받을 수 있다.**  

3. **US-QNA-03 (P1 / Should)**

   * As a user,
   * I want **답변이 유용했는지 간단히 평가(좋아요/별로예요)** 할 수 있고,
   * so that **추후 시스템이 답변 품질을 개선하는 데 기여**할 수 있다. 

---

### 4.6 Epic – 계정 & 세션

**E-AUTH**

1. **US-AUTH-01 (P1 / Must)**

   * As a user,
   * I want **검색/상세/Q&A 같은 기본 기능은 로그인 없이도 사용**할 수 있고,
   * so that **서비스 진입 장벽 없이 먼저 경험**해 볼 수 있다. 

2. **US-AUTH-02 (P1 / Must)**

   * As a user,
   * I want **레시피북 저장/보정 레시피 저장 시점에만 로그인/회원가입을 요구**받고,
   * so that **필요할 때 자연스럽게 계정을 만들 수 있다.** 

3. **US-AUTH-03 (P1 / Must)**

   * As a user,
   * I want **로그인 후 방금 보던 화면으로 자연스럽게 돌아와 저장/보정 작업이 계속 이어지길** 원하며,
   * so that **가입으로 인한 흐름 끊김 없이 Core Loop를 완주**할 수 있다.

### 4.7 Epic – 유저 레시피 게시 & 피드 (Phase 2+ 참고)

**E-POST (Future)** 

1. **US-POST-01 (P2)**

   * As a 파워 유저/크리에이터,

   * so that **복잡한 에디팅 없이도 내 레시피를 공유**할 수 있다.

2. **US-POST-02 (P2)**

   * As a 크리에이터,
   * I want **내 레시피가 저장/보정/재조리되는 데이터를 볼 수 있는 대시보드**를 갖고,
   * so that **내 레시피를 개선하고 팬과 더 잘 소통**할 수 있다.

## 5. 활용 방법

* **디자인**: 각 시나리오(SC-01~04)를 기준으로 사용자 흐름(Flow Chart), 화면 설계를 구체화한다.
* **개발/QA**: User Story ID(US-SEARCH-01 등)를 이슈 트래킹 시스템 티켓과 매핑하여,

  * 요구사항 정의서/기능 명세서의 세부 조건을 **Acceptance Criteria**로 연결해 구현·테스트 기준으로 활용한다.  
* **데이터/그로스**: 각 시나리오가 실제로 얼마나 수행되는지

  * Core Loop 완주율, 검색→상세 전환율, 보정 레시피 재사용률 등의 KPI와 직접 연결하여 모니터링한다. 

---

필요하다면, 다음 단계에서 **각 User Story별 세부 Acceptance Criteria 테이블**이나 **화면별 시퀀스 다이어그램**도 따로 분리해서 만들어 줄 수 있다.
